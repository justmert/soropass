A **passkey** is the credential your device already uses for Face ID, Touch ID,
Windows Hello, or a hardware security key. Instead of a 24-word seed phrase, the
private key is generated and stored by the device's secure hardware and never
leaves it — the user just confirms with a fingerprint or face scan.

This SDK lets a **Stellar smart account** (a Soroban contract account) be
controlled by a passkey, and ships as a drop-in module for
[Stellar Wallets Kit](https://github.com/Creit-Tech/Stellar-Wallets-Kit).

## Why passkeys

- **Nothing to lose or phish.** There is no seed phrase; the key lives in the
  device's secure enclave.
- **Familiar UX.** Users already know Face ID / Touch ID / Windows Hello.
- **Synced and recoverable.** Platform passkeys sync via iCloud Keychain and
  Google Password Manager, so the account follows the user across their devices.
- **Verified on-chain.** Soroban smart accounts check the passkey's secp256r1
  signature directly in the contract — no trusted server in the loop.

## How it works

<Steps>
  <Step title="The passkey signs">
    The browser's WebAuthn API produces a secp256r1 signature using the device key.
  </Step>
  <Step title="The SDK assembles the authorization">
    It normalizes the signature and builds the Soroban authorization the contract expects.
  </Step>
  <Step title="The smart account verifies it on-chain">
    The contract's `__check_auth` runs `secp256r1_verify` — the passkey is the account's signer.
  </Step>
</Steps>
