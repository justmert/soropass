The Recover screen is the lost-localStorage / new-device path: the user confirms with a
discoverable passkey, and the screen lists every smart account that passkey controls. It's
bound to a headless `RecoverFlow`, and the account list is a **real listbox** with full
keyboard support.

```ts
import { createRecoverFlow } from '@stellar-passkey/ui/headless';
import { mountRecoverScreen } from '@stellar-passkey/ui/styled';
import '@stellar-passkey/ui/styled.css';
```

## States

`RecoverFlowState` is a six-variant union тАФ the only screen with six states, because a single
passkey can control zero, one, or many accounts.

| State | What the user sees | Busy look |
| --- | --- | --- |
| `idle` | Header + a **Recover** button | тАФ |
| `discovering` | Idle content dimmed behind the **calm opaque OS-sheet panel** + pulsing glyph, no spinner | OS sheet |
| `resolved` | A title (`"N accounts found"` or `"Account found"`) + the **account listbox** + **Continue** (disabled until a row is selected) | тАФ |
| `selected` | The same listbox with one row highlighted (`aria-selected`, brand highlight, check icon) + **Continue** enabled | тАФ |
| `none` | A muted glyph, "No accounts found", and **Create a new passkey instead** / **Try a different passkey** | тАФ |
| `error` | The single error layout тАФ title + assertive message + **Try again** / **Retry** | тАФ |

<Note>
  Empty discovery (0 accounts) goes to `none`, not `resolved`. The styled layer caches the
  resolved account list so the `selected` state (which carries only the chosen account) still
  renders the full highlighted list.
</Note>

## Usage

```ts
import { recover } from '@stellar-passkey/core/recover';

const flow = createRecoverFlow({
  recover: () => recover({ rpId: 'wallet.example.com', indexer }), // тЖТ RecoverResult[]
});

const handle = mountRecoverScreen(document.getElementById('root')!, {
  flow,
  accountMeta: (_account, i) => `Recovered account ${i + 1}`,
  onContinue: (account) => connect(account.contractId),
  onCreateNew: () => router.push('/create'),
});
// later: handle.unmount();
```

<Note>
  `recover` returns an **array** тАФ one credential can control multiple accounts. The screen
  handles 0 (`none`), 1 (`"Account found"`), and many (`"N accounts found"`) automatically.
</Note>

### Via the kit

Through `@creit.tech/stellar-wallets-kit`, the recover ceremony still runs on the SDK; the
screen wraps it the same way:

```ts
const flow = createRecoverFlow({
  recover: () => recover({ rpId, indexer }),
});
mountRecoverScreen(root, { flow, onContinue: (a) => kit.setWallet(/* select the account */) });
```

See [Wallets Kit integration](/wallets-kit).

## Props тАФ `RecoverScreenOptions`

<ParamField path="flow" type="RecoverFlow" required>
  The headless flow from `createRecoverFlow({ recover })`.
</ParamField>

<ParamField path="copy" type="Partial<RecoverCopy>">
  Override any of the 16 copy fields; defaults are `DEFAULT_RECOVER_COPY`.
</ParamField>

<ParamField path="accountMeta" type="(account: RecoverResult, index: number) => string">
  Renders the meta line under each account row. Defaults to `"Stellar smart account"`.
</ParamField>

<ParamField path="onContinue" type="(account: RecoverResult) => void">
  Called from **Continue** with the selected account (`{ contractId, credentialId }`).
</ParamField>

<ParamField path="onCreateNew" type="() => void">
  Called from "Create a new passkey instead" in the `none` state тАФ host navigates to Create.
</ParamField>

### Returns

<ResponseField name="ScreenHandle" type="{ unmount(): void }">
  Unsubscribes from the flow and clears the root.
</ResponseField>

## Accessibility тАФ the account listbox

The recover list is a genuine `<ul role="listbox">` with `<li role="option">` rows. The
headless layer ships the roles + selection; the styled layer adds roving focus and the full
keyboard pattern.

| Key | Action |
| --- | --- |
| `ArrowDown` / `ArrowUp` | Move the active option (wraps at the ends) |
| `Home` / `End` | Jump to first / last option |
| `Enter` / `Space` | Commit the active option as the selection |
| Click | Same as Enter on that row |

- **Roving tabindex** тАФ only the active row has `tabIndex=0`; the rest are `-1`. Arrow keys
  move focus locally without a re-render.
- **Selection** тАФ the selected row gets `aria-selected="true"` + a brand highlight + a check
  icon.
- **Focus on `resolved`** тАФ focus lands on the **first account row** (`[data-pk-focus]`). On
  `none` / `error`, focus lands on the message paragraph. Roving focus inside the listbox is
  **not** disturbed by re-renders тАФ only by a status change.
- **Each row** shows an identicon + middle-truncated address + the `accountMeta` line.
