## Install

```bash
npm install @stellar-passkey/core @stellar/stellar-sdk
```

<Note>
  `@stellar/stellar-sdk` (v15+) is a **peer dependency** — bring your own; it is
  never bundled into the SDK.
</Note>

You provide two things that are specific to your deployment:

- a **smart-account factory** (`deployer`) that deploys an account for a public key, and
- an **RPC URL** (for example, Stellar testnet).

## Create a wallet

The browser shows the native passkey sheet; the user confirms with Face ID /
Touch ID. Only secp256r1 (ES256) keys are accepted — that's what Stellar verifies
on-chain — so unsupported keys are rejected automatically.

```ts
import { createPasskey } from '@stellar-passkey/core/create';

const account = await createPasskey({
  rpId: 'wallet.example.com', // your site's domain
  rpName: 'Example Wallet',
  userName: 'alice',
  userActivation: navigator.userActivation,
  deployer: {
    deploy: ({ publicKey, credentialId }) => myFactory.deploy(publicKey, credentialId),
  },
});

account.contractId; // the smart-account address, e.g. "C…"
account.credentialId; // the passkey id — store it to reconnect later
```

## Sign a transaction

<Steps>
  <Step title="Reconnect the account">
    ```ts
    import { connect } from '@stellar-passkey/core/connect';
    import { eventsIndexer } from '@stellar-passkey/core';

    const indexer = eventsIndexer({ rpcUrl, factoryContractId });
    const session = await connect({ rpId, indexer }); // { contractId, credentialId }
    ```
  </Step>
  <Step title="Sign with the passkey">
    Build and simulate your transaction with `@stellar/stellar-sdk` first (to fill
    in fees and nonces), then sign its authorization with the passkey:

    ```ts
    import { signTransaction, browserPasskeySigner } from '@stellar-passkey/core/sign';

    const sign = browserPasskeySigner({ rpId, allowCredentials: [session.credentialId] });
    const signedXdr = await signTransaction(preparedTxXdr, { networkPassphrase, sign });
    ```
  </Step>
  <Step title="Submit">
    ```ts
    import { directSubmission } from '@stellar-passkey/core';

    const result = await directSubmission({ rpcUrl, networkPassphrase }).send(fullySignedXdr);
    // result.status: 'SUCCESS' | 'FAILED' | 'PENDING'
    ```
  </Step>
</Steps>

<Tip>
  Building a wallet? The [Stellar Wallets Kit guide](/wallets-kit) wraps all of
  this so the passkey works like any other wallet option — usually the better
  starting point.
</Tip>

## Recover on a new device

A passkey can be discoverable, so a returning user finds their accounts with no
stored state:

```ts
import { recover } from '@stellar-passkey/core/recover';

const accounts = await recover({ rpId, indexer }); // [{ contractId, credentialId }]
// 0 → offer to create one; 1 → use it; many → let the user choose.
```

## Handle errors

Every failure is a `KitError` with a stable `code` you can map to UI copy:

```ts
import { isKitError } from '@stellar-passkey/core/types';

try {
  await createPasskey(/* … */);
} catch (e) {
  if (isKitError(e)) {
    // 'USER_CANCELLED' | 'UNSUPPORTED_AUTHENTICATOR' | 'ES256_NOT_SUPPORTED'
    // | 'INVALID_PUBLIC_KEY' | 'CONTRACT_AUTH_FAILED' | 'NETWORK_ERROR' | …
  }
}
```

See every function, option, and error in the [API reference](/api-reference).
