---
title: "createPasskey"
description: "Register an ES256-only passkey and deploy a Stellar smart account in a single ceremony."
---

`createPasskey` runs the registration ceremony: it advertises **only** ES256,
creates the credential, extracts the SEC-1 public key (a non-ES256 key
hard-fails here), deploys the smart account via your factory, persists the
credential id, and returns the account.

```ts
import { createPasskey } from '@stellar-passkey/core/create';
```

## Usage

The `deployer` is contract-specific — you supply the factory call. The SDK never
reinvents the contract layer; it just hands your deployer the new credential's
public key and id.

<CodeGroup>

```ts Standalone
import { createPasskey } from '@stellar-passkey/core/create';

const account = await createPasskey({
  rpId: 'wallet.example.com',          // your registrable domain
  rpName: 'Example Wallet',
  userName: 'alice',
  userActivation: navigator.userActivation, // enforces the Safari gesture rule
  deployer: {
    async deploy({ publicKey, credentialId }) {
      // your factory call — kalepail webauthn-wallet / OZ Smart Account, etc.
      const { contractId, txHash } = await myFactory.deploy(publicKey, credentialId);
      return { contractId, txHash };
    },
  },
});

console.log(account.contractId);   // 'C…' smart-account address
console.log(account.credentialId); // base64url credential id — persist / pass to the signer
console.log(account.publicKey);    // Uint8Array(65) — SEC-1 uncompressed P-256 key
```

```ts Via the Kit
// With the PasskeyModule registered, create runs through the module's
// createAccount() helper rather than being called directly.
import { PasskeyModule } from '@creit.tech/stellar-wallets-kit/modules/passkey';

const passkey = new PasskeyModule({
  rpId: 'wallet.example.com',
  rpName: 'Example Wallet',
  networkPassphrase,
  deployer,
  indexer,
});

const credential = await passkey.createAccount('alice'); // -> PasskeyCredential
```

</CodeGroup>

<Warning>
**Call `createPasskey` inside a click/tap handler.** Safari/WebKit silently
reject `navigator.credentials.create` outside a transient user activation. Pass
`navigator.userActivation` so the SDK enforces it; otherwise the ceremony fails
with `USER_CANCELLED`. Never auto-trigger it from a mount or effect.
</Warning>

## Parameters

`createPasskey(options: CreatePasskeyOptions)`

<ParamField body="rpId" type="string" required>
  The Relying Party ID — your site's registrable domain (e.g.
  `wallet.example.com`). The passkey is scoped to this domain.
</ParamField>

<ParamField body="rpName" type="string" required>
  Human-readable name shown in the OS passkey sheet.
</ParamField>

<ParamField body="userName" type="string" required>
  The account's user name (e.g. an email or handle). Also used as the default
  `userId` (its UTF-8 bytes) when one is not supplied.
</ParamField>

<ParamField body="deployer" type="AccountDeployer" required>
  Deploys the smart account for the new passkey. You implement
  `deploy({ publicKey, credentialId }): Promise<{ contractId, txHash? }>` with
  your factory call.
</ParamField>

<ParamField body="userActivation" type="{ isActive: boolean }">
  Pass `navigator.userActivation` to enforce the Safari/WebKit user-gesture
  rule. Omitted ⇒ lenient (kept isomorphic for non-browser/older engines).
</ParamField>

<ParamField body="webauthn" type="WebAuthnClient">
  Abstraction over `navigator.credentials`. Defaults to
  `browserWebAuthnClient()`. Inject a custom client for tests / non-browser.
</ParamField>

<ParamField body="storage" type="CredentialStorage">
  Per-`rpId` credential-id persistence. Defaults to `defaultCredentialStorage()`
  (localStorage, with an in-memory fallback for Node/SSR).
</ParamField>

<ParamField body="userId" type="Uint8Array">
  Stable, opaque user handle. Defaults to the UTF-8 bytes of `userName`.
</ParamField>

<ParamField body="challenge" type="Uint8Array">
  Registration challenge. Defaults to 32 cryptographically random bytes.
</ParamField>

<ParamField body="residentKey" type="'discouraged' | 'preferred' | 'required'" default="required">
  Defaults to `'required'` so the passkey is **discoverable** — this is what
  makes [`recover`](/sdk/recover) work on a new device. `requireResidentKey` is
  set to `true` when this is `'required'`.
</ParamField>

<ParamField body="userVerification" type="'discouraged' | 'preferred' | 'required'" default="preferred">
  Forwarded to the authenticator selection criteria.
</ParamField>

<Note>
`attestation` is always `'none'` — only the public key is needed, which avoids
attestation-statement parsing and the privacy surface that comes with it.
</Note>

## Returns

`Promise<PasskeyCredential>`

<ResponseField name="contractId" type="string">
  The Soroban smart-account `C…` address that was deployed.
</ResponseField>

<ResponseField name="credentialId" type="string">
  base64url credential id from the authenticator. Persist this (and pass it as
  `allowCredentials` to the signer) so the OS later offers the right passkey.
</ResponseField>

<ResponseField name="publicKey" type="Uint8Array">
  65-byte SEC-1 uncompressed P-256 public key.
</ResponseField>

## Errors

This ceremony can throw the following [`KitError`](/sdk/errors) codes:

| Code | When |
| --- | --- |
| `USER_CANCELLED` | OS passkey sheet dismissed, or called outside a user activation. |
| `ES256_NOT_SUPPORTED` | The authenticator returned a non-ES256 credential — hard-fail at public-key extraction. |
| `INVALID_PUBLIC_KEY` | Malformed COSE/CBOR, wrong key type/curve, or an off-curve P-256 point. |
| `UNSUPPORTED_AUTHENTICATOR` | No WebAuthn available, or a malformed/failed registration response. |

## ES256-only options

If you need the raw registration options (for a custom WebAuthn client), use
`buildCreateOptions` — it is the single source of the non-negotiable `alg: -7`
surface.

```ts
import { buildCreateOptions } from '@stellar-passkey/core/create';

const options = buildCreateOptions({
  rpId: 'wallet.example.com',
  rpName: 'Example Wallet',
  userName: 'alice',
  challenge: crypto.getRandomValues(new Uint8Array(32)),
});

options.pubKeyCredParams;                       // [{ type: 'public-key', alg: -7 }] — ES256 and ONLY ES256
options.authenticatorSelection.residentKey;     // 'required' — discoverable by default
options.attestation;                            // 'none'
options.timeout;                                // 60000
```
