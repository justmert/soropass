title: "Adapters"
description: "Two small interfaces — SubmissionAdapter and IndexerAdapter — are the SDK's only infra coupling points. Zero-infra by default, one-line to swap."
---

Submission (how a signed transaction reaches the network) and indexing
(resolving a credential id → smart-account address) are the **only** infra
coupling points in the SDK. They are expressed as two small interfaces, so the
ceremonies depend on a shape, never a concrete backend. The default stack needs
only an RPC URL — no relayer, no API key.

```ts
// Interfaces come from /types; factories from the umbrella root:
import type { SubmissionAdapter, IndexerAdapter, SubmitResult, ResolvedAccount } from '@stellar-passkey/core/types';
import { defaultAdapters, directSubmission, eventsIndexer } from '@stellar-passkey/core';
```

<Note>
Submission/indexer infrastructure is **untrusted** by design. A malicious or
buggy relayer/indexer cannot forge an authorization — it can only withhold,
delay, or mis-report a lookup. That is exactly why they are pluggable, not
hard-coupled. See [Security](/security).
</Note>

## The interfaces

### SubmissionAdapter

<ParamField path="send" type="(signedTxXdr: string) => Promise<SubmitResult>" required>
  Send a signed base64-XDR transaction to the network and report the result.
</ParamField>

`SubmitResult`:

<ResponseField name="status" type="'SUCCESS' | 'PENDING' | 'FAILED'">
  The outcome of submission.
</ResponseField>
<ResponseField name="hash" type="string">
  Transaction hash (hex), when known.
</ResponseField>
<ResponseField name="returnValue" type="unknown">
  Decoded contract return value on success (implementation-defined).
</ResponseField>
<ResponseField name="errorResultXdr" type="string">
  base64 XDR of the failure result, when failed.
</ResponseField>

### IndexerAdapter

<ParamField path="resolveByCredential" type="(credentialId: string) => Promise<ResolvedAccount[]>" required>
  Resolve a credential id to the smart accounts it controls.
</ParamField>

`ResolvedAccount`:

<ResponseField name="contractId" type="string">
  A Soroban smart-account `C…` address.
</ResponseField>

## Default zero-infra stack

`defaultAdapters` wires the `direct` submission + `events` indexer pair — no
external services.

```ts
import { defaultAdapters } from '@stellar-passkey/core';

const { submission, indexer } = defaultAdapters({
  rpcUrl: 'https://soroban-testnet.stellar.org',
  networkPassphrase: 'Test SDF Network ; September 2015',
  factoryContractId: 'C…FACTORY',
});

const result = await submission.send(signedXdr);
if (result.status === 'SUCCESS') console.log('tx', result.hash, result.returnValue);
```

<ParamField body="rpcUrl" type="string" required>
  Soroban RPC endpoint.
</ParamField>
<ParamField body="networkPassphrase" type="string" required>
  Network passphrase (e.g. testnet).
</ParamField>
<ParamField body="factoryContractId" type="string" required>
  The passkey factory contract whose deploy events the indexer scans.
</ParamField>
<ParamField body="allowHttp" type="boolean">
  Allow `http://` RPC URLs. Defaults to `true` when `rpcUrl` starts with `http://`.
</ParamField>

## Adapter catalog

| Role | Adapter | Default? | Use it for |
| --- | --- | --- | --- |
| Submission | `directSubmission` | **yes** | Zero-infra: send straight to soroban-rpc and poll. No relayer, no key. |
| Submission | `launchtubeSubmission` | optional | Legacy Launchtube relay (invite/credit-gated). |
| Submission | `openzeppelinRelayerSubmission` | optional | OZ Relayer / Channels — the post-Launchtube direction. |
| Indexer | `eventsIndexer` | **yes** | Zero-infra: soroban-rpc `getEvents` filtered on the factory contract. |
| Indexer | `mercuryIndexer` | optional | Mercury / Zephyr indexing service. |

Swapping any adapter is a one-line config change — same interface, zero ceremony
changes.

### directSubmission

```ts
import { directSubmission } from '@stellar-passkey/core';

const submission = directSubmission({
  rpcUrl: 'https://soroban-testnet.stellar.org',
  networkPassphrase: 'Test SDF Network ; September 2015',
});
```

Sends via `server.sendTransaction(tx)` then polls with `server.pollTransaction`.
An `ERROR` send maps to `FAILED` with the `errorResultXdr`.

<ParamField body="rpcUrl" type="string" required />
<ParamField body="networkPassphrase" type="string" required />
<ParamField body="allowHttp" type="boolean" />

### eventsIndexer

```ts
import { eventsIndexer } from '@stellar-passkey/core';

const indexer = eventsIndexer({
  rpcUrl: 'https://soroban-testnet.stellar.org',
  factoryContractId: 'C…FACTORY',
});
```

Scans `getEvents` filtered on the factory contract. By default it scans ~1 day
back (`17280` ledgers ≈ one day at 5s ledgers) and matches events whose XDR
contains the credential id.

<ParamField body="rpcUrl" type="string" required />
<ParamField body="factoryContractId" type="string" required>
  The factory that emits a deploy/add event per account.
</ParamField>
<ParamField body="startLedger" type="number">
  Ledger to scan from. Defaults to ~1 day back from the latest ledger.
</ParamField>
<ParamField body="allowHttp" type="boolean" />
<ParamField body="matchEvent" type="(event, credentialId) => ResolvedAccount | null">
  Override the event → account mapping. The default is best-effort; wire the
  exact factory event schema here for your deployed contract.
</ParamField>

### launchtubeSubmission (optional)

```ts
import { launchtubeSubmission } from '@stellar-passkey/core';

const submission = launchtubeSubmission({ url: 'https://launchtube.example', token: '…' });
```

<ParamField body="url" type="string" required>
  Launchtube relay endpoint.
</ParamField>
<ParamField body="token" type="string">
  Bearer token (Launchtube is invite/credit-gated).
</ParamField>

### openzeppelinRelayerSubmission (optional)

```ts
import { openzeppelinRelayerSubmission } from '@stellar-passkey/core';

const submission = openzeppelinRelayerSubmission({ url: 'https://relayer.example', apiKey: '…' });
```

<ParamField body="url" type="string" required>
  OZ Relayer / Channels endpoint.
</ParamField>
<ParamField body="apiKey" type="string">
  Bearer API key.
</ParamField>

### mercuryIndexer (optional)

```ts
import { mercuryIndexer } from '@stellar-passkey/core';

const indexer = mercuryIndexer({ url: 'https://mercury.example', token: '…' });
```

<ParamField body="url" type="string" required>
  Mercury / Zephyr endpoint.
</ParamField>
<ParamField body="token" type="string">
  Bearer token.
</ParamField>

  Map the Mercury/Zephyr JSON response to resolved accounts. Defaults to reading
  a `{ accounts: [{ contractId }] }` shape.
</ParamField>

## Swapping adapters

Because every adapter shares an interface, switching infra is a single line —
the ceremonies never change:

```ts
import { defaultAdapters, openzeppelinRelayerSubmission, mercuryIndexer } from '@stellar-passkey/core';

const { submission, indexer } = defaultAdapters({ rpcUrl, networkPassphrase, factoryContractId });

// Same SubmissionAdapter shape — swap submission for a managed relay:
const managedSubmission = openzeppelinRelayerSubmission({ url: 'https://relayer.example', apiKey });

// Same IndexerAdapter shape — swap events for Mercury:
const managedIndexer = mercuryIndexer({ url: 'https://mercury.example', token });
```

## Writing a custom adapter

Implement the interface — anything that satisfies the shape works:

```ts
import type { SubmissionAdapter, SubmitResult } from '@stellar-passkey/core/types';

const myQueueSubmission: SubmissionAdapter = {
  async send(signedTxXdr): Promise<SubmitResult> {
    const { hash } = await myQueue.enqueue(signedTxXdr);
    return { status: 'PENDING', hash };
  },
};
```

```ts
import type { IndexerAdapter, ResolvedAccount } from '@stellar-passkey/core/types';

const myDbIndexer: IndexerAdapter = {
  async resolveByCredential(credentialId): Promise<ResolvedAccount[]> {
    const rows = await db.accountsByCredential(credentialId);
    return rows.map((r) => ({ contractId: r.contractId }));
  },
};
```
