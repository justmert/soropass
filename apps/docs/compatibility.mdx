---
title: Browser compatibility
description: Where passkeys and the WebAuthn features this SDK relies on are supported.
---

Passkeys (WebAuthn) are supported across all modern browsers and platforms. The
table below covers the capabilities this SDK relies on. **ES256 (secp256r1)** —
the only algorithm Stellar verifies on-chain, and the one the SDK requires — is
supported everywhere listed.

| Browser / platform | Passkeys | Platform authenticator | Autofill (Conditional UI) | Cross-device (hybrid) | Multi-origin |
| --- | :---: | :---: | :---: | :---: | :---: |
| Chrome — desktop | ✅ | ✅ | ✅ | ✅ | ✅ |
| Chrome — Android | ✅ | ✅ | ✅ | ✅ | ✅ |
| Edge — Windows | ✅ | ✅ | ✅ | ✅ | ✅ |
| Safari — macOS | ✅ | ✅ | ✅ | ✅ | ❔ |
| Safari — iOS | ✅ | ✅ | ✅ | ✅ | ❔ |
| Firefox — desktop | ✅ | ✅ | ✅ | ❔ | ❌ |
| Firefox — Android | ✅ | ✅ | ✅ | ❔ | ❌ |
| Samsung Internet | ✅ | ✅ | ✅ | ✅ | ✅ |

✅ supported · ❔ partial / unverified · ❌ not supported

<Info>
  **What the columns mean**
  - **Passkeys** — `navigator.credentials.create()` / `.get()` (WebAuthn).
  - **Platform authenticator** — `isUserVerifyingPlatformAuthenticatorAvailable()`; whether the device itself (Face ID / Touch ID / Windows Hello) can be the passkey.
  - **Autofill (Conditional UI)** — passkeys offered inline in the login field.
  - **Cross-device (hybrid)** — sign in on a desktop using your phone's passkey.
  - **Multi-origin** — one passkey shared across related domains (Related Origin Requests).
</Info>

## Check the user's own device

Detect support at runtime before showing the passkey option:

```ts
const supported =
  typeof window !== 'undefined' &&
  !!window.PublicKeyCredential &&
  (await PublicKeyCredential.isUserVerifyingPlatformAuthenticatorAvailable());
```

In Stellar Wallets Kit this is handled for you — `PasskeyModule.isAvailable()`
returns `false` (within a 500&nbsp;ms budget) when the platform can't use passkeys,
so the kit can hide the option.

## How this stays current

This table isn't maintained by hand. Chromium and Edge are **re-verified
automatically** on every release and weekly, by running real passkey ceremonies
against a virtual authenticator and checking the signatures. Safari, Firefox, and
mobile — which can't be fully automated — are re-checked on real devices monthly
and after major OS/browser releases. Each run is recorded and dated, so support
changes are visible over time rather than going stale.
