🏭Soropass — Account Creation & Discovery
One passkey maps deterministically to one Soroban smart-account address. Creation deploys it; recovery re-derives it by reading a public on-chain event — no Soropass database ever holds the mapping.
Create a new account 🆕 first time
1
Create the passkey

createPasskey() → ES256 credential: credentialId + P-256 public key (SEC-1, 65 bytes).

2
Factory deploy

AccountFactory.deploy(public_key, credential_id), salt = SHA256(credential_id) → a deterministic C-address.

3
Emit discovery event

Contract publishes ("deployed", credential_id) → address.

Reconnect / recover 🔁 any device
1
Discoverable assertion

connect() / recover() — the passkey returns its credentialId (resident credential).

2
Resolve via the indexer

eventsIndexer.resolveByCredential(id) reads soroban-rpc getEvents on the factory.

3
Same address, back online

The deployed event yields the same C-address → account reconnected.

🧭Deterministic mapping:one passkey ⇄ one C-address. Discovery is just reading a public event over public RPC — zero-infra, no custodial lookup service. Mercury is an optional drop-in indexer.
